# QUESTÃO 2 (questao 5) 

- Criei um arquivo .c contendo a função dump que recebe um ponteiro 
de qualquer tipo e um inteiro que dirá para a questão o tamanho desse 
ponteiro, já que ele é um "ponteiro universal". Criei também outro 
arquivo .c contendo a main. Fiz a compilação de ambos os arquivos e 
executei, então o programa me retornou isso:

    valor de c1: 1 -> na memória: 1 
    valor de c2: 49 -> na memória: 49 
    valor de i: 1 -> na memória: 1 0 0 0 
    valor de v: 1 -> na memória: 49 0 

Percebi que o programa mostra como a variável está organizada na 
memória, conforme o tipo e conteúdo. Uma variável do tipo char 
ocupa 1 byte na memória, c1 contém o codigo 1 que equivale a 
algum caractere, já c2 contém o codigo do caractere '1' que é 49, 
ja a variavel i contém o inteiro 1, e está organizada "1 0 0 0" 
pois contém 4 bytes e está ao contrário, o certo seria "0 0 0 1" 
mostrando que minha maquina é little endian já que nessas maquinas 
o byte menos significativo vem primeiro. Já a variável v é uma 
lista de caracteres, ou em outras palavras uma string, onde cada 
caractere ocupa um espaço na string, nesse caso a string é o 
numero 1 mais o caractere \0, mostrando que ocupa 2 bytes na 
memória.
